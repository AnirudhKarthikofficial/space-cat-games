The shift to React.js, and changes in branching and committing.


Yeah, this is an adhoc page. But it's here to explain the changes in branching and committing. Currently, all changes to be deployed are pushed to main branch. This will not change, however unused branches like minimal and draft/summer-brook will be deleted. The current working tree before the shift will be stored in a branch called pre-react. This branch will not be deleted, and will be used to revert to if necessary. The new working tree will live in main. The react branch will be archived. Happy coding! - Meowcat767 <3
